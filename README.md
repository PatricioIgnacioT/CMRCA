# CMRCA

CMRCA (Contact-based Metaparticle Reconstruction and Characterization Algorithm) is a MATLAB post-processing tool for DEM simulations involving bonded particles.

The algorithm identifies metaparticles from particle and contact data, reconstructs their geometry, and exports particle- and metaparticle-level information for further analysis.

## Description

CMRCA was developed to post-process DEM simulations exported from Rocky DEM. The code uses particle and contact CSV files to identify groups of bonded particles that form each metaparticle. Then, it estimates the characteristic size of each metaparticle, calculates its center coordinates, determines its main local dimensions, and exports the results to an Excel file.

The algorithm also generates STL files representing the reconstructed metaparticles.

## Required files for the main algorithm

The following MATLAB files are required to run the main CMRCA algorithm:

* `script_algorithm_CMRCA.m`: main script.
* `Read_Particles.m`: function used to read particle CSV files.
* `Read_Contacts.m`: function used to read contact CSV files.
* `findFarthestPoints.m`: function used to identify the two farthest points of each metaparticle.

## Optional files for granulometry post-processing

The files inside the `optional_granulometry` folder are not required to run the main CMRCA algorithm.

They are only used if the user wants to calculate and plot the metaparticle size distribution from the Excel file generated by CMRCA.

* `granulometry.m`: script used to calculate and plot the cumulative mass passing curve.
* `read_xlsx.m`: function used by `granulometry.m` to read numeric data from the Excel output file.

Therefore, CMRCA can be executed without `granulometry.m` and `read_xlsx.m`. These files are only complementary tools for granulometry analysis.

## User inputs

Before running the main script, the user must edit the paths in `script_algorithm_CMRCA.m`.

```matlab
particles_folder = 'C:\Users\YOUR_USER\Documents\YOUR_FOLDER\Particles'; % Folder containing the particle CSV files exported from Rocky.
contacts_folder = 'C:\Users\YOUR_USER\Documents\YOUR_FOLDER\Contacts'; % Folder containing the contact CSV files exported from Rocky.
output_folder = 'C:\Users\YOUR_USER\Documents\YOUR_FOLDER\Results'; % Folder where the script saves the STL files and Excel results.
```

These paths must be correctly defined. The script will not work if the folders do not exist or if they do not contain the required CSV files.

The user must also define the time range to be processed:

```matlab
initial_time = 125; % First time step to process.
final_time = 125; % Last time step to process.
```

The file prefixes must match the names of the CSV files exported from Rocky DEM:

```matlab
particle_file_prefix = 'Particles_bpm'; % Prefix of the particle CSV files before the six-digit time number.
contact_file_prefix = 'Contacts_Compresion_bpm'; % Prefix of the contact CSV files before the six-digit time number.
```

For example, if the particle file is named:

```text
Particles_bpm000125.csv
```

then the particle prefix must be:

```matlab
particle_file_prefix = 'Particulas_bpm';
```

## Input files

The main algorithm requires:

* Particle CSV files exported from Rocky DEM.
* Contact CSV files exported from Rocky DEM.


## Outputs

CMRCA exports:

* An Excel file named `Post_results.xlsx`.
* STL files for the reconstructed metaparticles.

The Excel file includes:

* Particle ID.
* Particle mass.
* Particle position.
* Particle velocity.
* Particle size.
* Metaparticle size.
* Metaparticle ID.
* Metaparticle center coordinates.


## Granulometry analysis

The particle size distribution does not need to be plotted manually. If the user wants to obtain the granulometry from the Excel output, the repository includes the `optional_granulometry` folder.

This folder contains `granulometry.m`, which reads the Excel file generated by CMRCA and plots the cumulative mass passing curve. The function `read_xlsx.m` is required only for this granulometry script.
